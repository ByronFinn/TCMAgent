# 红旗征象筛查协议（Red Flags Screening Protocol）

> **Skill 用途**：本文件定义 `triage-agent` 在导诊阶段必须执行的红旗征象筛查协议。红旗筛查是整个问诊流程中**优先级最高**的环节，必须在任何收敛式追问开始之前完成。

---

## 1. 核心原则

1. **安全优先于效率**：宁可多问一个红旗问题，不可因追求流程效率而跳过安全筛查。
2. **严重红旗立即中断**：一旦识别任何严重红旗（Level 1），必须立即停止普通问诊，进入安全处置流程。
3. **不依赖患者主动汇报**：患者不一定知道哪些症状是危险信号，`triage-agent` 必须主动筛查。
4. **先筛严重，再筛中度**：按照 Level 1 → Level 2 → Level 3 的顺序依次评估，不可跳序。
5. **红旗记录不可删除**：一旦记录到 `red_flags`，不得因患者后续否认而删除，应追加矛盾记录。

---

## 2. 严重红旗（Level 1）— 立即停止线上问诊

以下任何一条满足，必须：
- 立即停止普通问诊流程
- 将 `risk_level` 设置为 `critical`
- 将 `safe_to_continue` 设置为 `false`
- 将 `recommended_route` 设置为 `emergency`
- 调用 `issue_risk_decision` 工具输出风险决策
- 向患者明确提示立即就医或拨打急救电话

### 2.1 心肺紧急

| 红旗 | normalized_key | 识别关键词 | 说明 |
|------|----------------|-----------|------|
| 胸痛伴呼吸困难 | `chest_pain` + `shortness_of_breath` | 胸口痛+喘不过气、胸闷憋气 | 可能：急性心梗、肺栓塞、气胸 |
| 静息性胸痛 | `chest_pain_at_rest` | 不动也胸痛、持续胸痛 | 高度警惕急性冠脉综合征 |
| 突发严重呼吸困难 | `sudden_severe_dyspnea` | 突然喘不上来气、呼吸费力 | 可能：急性哮喘发作、心衰、气胸 |
| 心悸伴晕厥/黑矇 | `palpitations` + `syncope` | 心跳很快然后眼前发黑、差点晕过去 | 可能：严重心律失常 |

### 2.2 神经系统紧急

| 红旗 | normalized_key | 识别关键词 | 说明 |
|------|----------------|-----------|------|
| 意识障碍 | `altered_consciousness` | 神志不清、叫不醒、反应迟钝、神志混乱 | 可能：颅内病变、严重感染、代谢危象 |
| 抽搐发作 | `seizure` | 抽搐、抽风、全身抖动不受控制、癫痫发作 | 需立即处置，不可远程观察 |
| 突发剧烈头痛 | `thunderclap_headache` | 突然头痛剧烈、"一生中最严重的头痛"、从没这么痛过 | 警惕蛛网膜下腔出血 |
| 卒中症状（FAST） | `stroke_symptoms` | 一侧脸歪、一只手臂无力、说话含糊/口齿不清、突然眼歪嘴斜 | 时间窗极其关键，立即拨 120 |
| 突发视力丧失 | `sudden_vision_loss` | 突然看不见、眼前全黑 | 可能：视网膜中央动脉阻塞等眼科急症 |

### 2.3 出血与感染紧急

| 红旗 | normalized_key | 识别关键词 | 说明 |
|------|----------------|-----------|------|
| 呕血 | `hematemesis` | 吐血、吐出来是红色的/咖啡色的 | 消化道大出血 |
| 黑便（柏油样） | `melena` | 大便是黑色的、沥青色、发臭 | 上消化道出血 |
| 便血（大量鲜血） | `hematochezia_severe` | 大便有很多血、厕所里都是血 | 下消化道出血 |
| 高热不退（>39°C） | `high_fever_persistent` | 发烧超过39度、高烧退不下去、烧了超过3天 | 警惕严重感染、脓毒血症 |
| 脑膜刺激征相关 | `neck_stiffness_fever` | 高烧+脖子僵硬不能弯、怕光怕声 | 警惕脑膜炎 |

### 2.4 其他立即处置

| 红旗 | normalized_key | 识别关键词 | 说明 |
|------|----------------|-----------|------|
| 严重过敏反应 | `anaphylaxis` | 突然全身起疹子+呼吸困难+脸肿、吃完东西喉咙肿 | 过敏性休克前兆 |
| 严重腹痛（撕裂/刀割样） | `severe_acute_abdomen` | 肚子突然剧烈疼痛、疼得弯不下腰、板状腹 | 可能：腹主动脉瘤破裂、消化道穿孔 |
| 妊娠急症相关 | `obstetric_emergency` | 孕期大出血、孕期剧烈腹痛、胎动消失 | 妇产科急诊，不适合线上 |

## 3. 中度风险标志（Level 2）— 建议转诊，可先采集信息

以下情况满足时：
- 将 `risk_level` 设置为 `high`
- `safe_to_continue` 可暂维持 `true`，但需在每轮问诊后重新评估
- 将 `recommended_route` 设置为 `offline_referral`
- 明确告知患者建议尽快线下就医
- 继续采集信息但不进入深度收敛推理

### 3.1 症状持续时间超限

| 情况 | 说明 |
|------|------|
| 发热持续 3–7 天无缓解 | 普通感冒不应超过7天，需排除其他感染 |
| 慢性咳嗽 > 3 周 | 需排除结核、肺癌、百日咳 |
| 不明原因体重下降 > 5kg/月 | 警惕恶性肿瘤、慢性感染、内分泌疾病 |
| 腹痛持续超过 48 小时 | 需线下腹部检查 |

### 3.2 系统性警示症状

| 情况 | normalized_key | 说明 |
|------|----------------|------|
| 夜间盗汗 + 体重下降 + 持续低热 | `night_sweating` + `weight_loss` + `low_grade_fever` | 结核/淋巴瘤警惕组合 |
| 进行性吞咽困难 | `progressive_dysphagia` | 需排除食管占位 |
| 无痛性肿块发现 | `painless_mass` | 需排除恶性肿瘤 |
| 尿血（无疼痛） | `painless_hematuria` | 需排除泌尿系肿瘤 |
| 便血 + 体重下降 | `blood_in_stool` + `weight_loss` | 需排除结直肠癌 |

### 3.3 神经系统非急性警示

| 情况 | 说明 |
|------|------|
| 进行性加重的头痛（数周内持续恶化） | 需排除颅内占位 |
| 新发癫痫（首次发作，非已知癫痫患者） | 需神经科评估 |
| 进行性肢体无力或感觉障碍 | 需排除中枢或外周神经系统病变 |

### 3.4 高风险既往病史 + 当前症状组合

| 组合 | 说明 |
|------|------|
| 已知心脏病史 + 胸闷/胸痛 | 风险显著上升，建议线下心电图 |
| 已知糖尿病 + 高热/伤口感染 | 感染进展风险高 |
| 已知慢性肾病 + 少尿/无尿 | 可能急性肾功能恶化 |
| 长期使用免疫抑制剂 + 发热 | 机会性感染风险高 |
| 抗凝药使用中 + 出血表现 | 出血风险需线下评估 |

## 4. 低风险但需标记（Level 3）— 继续线上，标记追踪

以下情况不中断问诊，但需写入 `red_flags` 并在推理中保持关注：

| 情况 | 处理 |
|------|------|
| 症状超过 2 周但患者自述缓解中 | 标记，继续问诊，总结时建议随诊 |
| 患者年龄 > 65 岁且症状模糊 | 标记老年人群，提高风险权重 |
| 患者提及"之前有类似发作"未完全恢复 | 标记复发风险，询问既往诊断 |
| 存在多种药物使用但患者说不清楚药名 | 标记用药不明，建议带药就诊 |
| 情志严重异常（强烈自杀或自伤倾向） | 立即升级至 Level 1，建议心理危机干预 |

> ⚠️ **特别说明**：自杀/自伤意念一经发现，立即升级为 Level 1，`safe_to_continue=false`，必须引导至专业心理援助或急诊。

## 5. 红旗筛查执行逻辑

### 5.1 标准筛查流程

```
Step 1：读取 case_state（主诉、已知症状、患者档案）
    ↓
Step 2：扫描 Level 1 关键词
    ├─ 命中任意 Level 1 → 立即进入 [严重红旗处置流程] → 停止
    └─ 未命中 → 继续
    ↓
Step 3：扫描 Level 2 关键词
    ├─ 命中 1 项 → risk_level=high，标记 recommended_route=offline_referral，继续采集
    └─ 未命中 → 继续
    ↓
Step 4：扫描 Level 3 情况
    ├─ 命中 → 写入 red_flags，设置追踪标记，继续问诊
    └─ 未命中 → 继续正常流程
    ↓
Step 5：完成筛查，更新 case_stage → triaged
```

### 5.2 严重红旗处置流程（Level 1 命中后）

```
1. 调用 screen_red_flags 工具，记录命中的红旗详情到 red_flags 字段
2. 调用 issue_risk_decision 工具：
   - risk_level = "critical"
   - safe_to_continue = false
   - recommend_offline_visit = true（建议急诊）
   - recommended_route = "emergency"
   - decision_reason = [具体红旗描述]
3. 调用 set_case_stage，将 case_stage 设置为 "handoff_required"
4. 输出患者可见提示（见第 6 节），停止所有普通问诊逻辑
```

### 5.3 筛查触发时机

红旗筛查应在以下时机执行：

| 时机 | 说明 |
|------|------|
| **建档初始化后立即执行** | 根据主诉内容做首次扫描 |
| **每轮问答后重新扫描** | 新症状可能触发红旗 |
| **患者主动提及新症状时** | 不等下一轮，即时扫描 |
| **风险复核阶段** | `safety-agent` 做最终全量复查 |

## 6. 患者可见提示模板

### 6.1 Level 1 严重红旗触发提示

```
您描述的症状中，有一些需要立即引起重视。

[具体症状描述，如：您提到的胸痛伴随呼吸困难]

这类情况可能提示需要紧急处理的状况，建议您：
• 立即停止当前问诊
• 拨打急救电话 120，或立即前往最近的急诊室
• 如身边有人，请让他人陪同就医

请不要等待或自行用药。您的安全是最重要的。
```

### 6.2 Level 2 中度风险提示

```
根据您描述的情况，我们注意到以下需要关注的问题：
[具体症状描述]

虽然目前不需要立即急诊，但建议您：
• 尽快（最好在 24–48 小时内）前往医院进行线下检查
• 在此之前，我们可以继续收集一些基本信息

如果症状突然加重，请立即前往急诊。
```

## 7. MatchedRedFlag 记录规范

每条命中的红旗应以 `MatchedRedFlag` 格式写入 `case_state.red_flags`：

| 字段 | 类型 | 说明 |
|------|------|------|
| `flag_id` | `str` | 唯一 ID，格式：`flag_{uuid4_short}` |
| `flag_name` | `str` | 红旗名称，如 `chest_pain_with_dyspnea` |
| `level` | `int` | 1 / 2 / 3 |
| `matched_facts` | `list[str]` | 触发该红旗的 `normalized_key` 列表 |
| `source_text` | `str` | 患者原始描述片段 |
| `detected_at_stage` | `str` | 检测时的 `case_stage` |
| `recommended_action` | `str` | 建议动作，如 `emergency` / `offline_referral` / `monitor` |
| `timestamp` | `str` | ISO 8601 时间戳 |

## 8. 红旗筛查注意事项

### 8.1 不可做的事

- ❌ 不可因患者否认严重症状就忽略——患者可能轻描淡写或不了解严重性
- ❌ 不可在 Level 1 红旗命中后继续普通问诊，哪怕患者坚持继续
- ❌ 不可删除已记录的红旗（即使患者后来说"其实没那么严重"）
- ❌ 不可将红旗问题与收敛性问题混合，红旗必须优先问完
- ❌ 不可自行判断"这个症状应该没问题"而不记录——交给 `issue_risk_decision` 工具

### 8.2 模糊情况处理

当患者描述不明确时（如"有点胸闷，不算很厉害"）：

1. 追问澄清：具体程度、持续时间、是否加重
2. 如无法澄清，**按更高风险等级处理**（保守原则）
3. 在 `red_flags` 中标记 `confidence` 为中等（0.5–0.7）
4. 在 `audit_log` 中记录"模糊症状，按偏高风险处理"

### 8.3 与特殊人群规则的联动

红旗筛查结果必须与特殊人群规则联动：
- **孕妇**：同等症状风险等级自动提升一级
- **儿童**（<12岁）：高热（>38.5°C）即应升为 Level 2
- **老年人**（>75岁）：模糊症状需按更高风险处理
- **慢性肾/肝病患者**：任何急性加重直接升为 Level 2

> 详见 `skills/triage/special-population-rules/SKILL.md`

## 9. 红旗问题话术参考

以下是 `triage-agent` 可参考的主动筛查话术（勿一次全问，按需选用）：

| 筛查目标 | 话术示例 |
|---------|---------|
| 胸痛/心脏 | "您有没有感觉胸口闷痛，或者喘不过气？" |
| 卒中 | "有没有突然感觉一侧手脚没力气，或者说话有点含糊？" |
| 意识 | "您目前神志清醒吗？身边有没有人？" |
| 出血 | "有没有吐血、大便发黑，或者其他出血的情况？" |
| 高热 | "有测体温吗？大概是多少度？高烧多久了？" |
| 头痛严重程度 | "这次头痛跟以往比，是不是特别剧烈，是那种突然非常非常痛的感觉？" |
| 妊娠 | "请问您目前是否怀孕，或者有可能怀孕？" |
| 抽搐 | "有没有出现过突然全身抖动、不受控制的情况？" |

## 10. 红旗筛查与问诊流程的边界

```
┌─────────────────────────────────────────┐
│           triage-agent 职责边界           │
│                                         │
│  ✅ 执行红旗筛查（本文件定义）             │
│  ✅ 记录命中的红旗到 case_state           │
│  ✅ 调用 issue_risk_decision 工具         │
│  ✅ 设置 case_stage 到适当值              │
│  ✅ 输出患者可见安全提示                   │
│                                         │
│  ❌ 不负责深度证型分析                    │
│  ❌ 不负责图谱候选生成                    │
│  ❌ 不负责动态追问收敛                    │
│  ❌ 不自行决定问诊路径（调工具决定）        │
└─────────────────────────────────────────┘
```

红旗筛查完成且无 Level 1 命中后，`triage-agent` 将控制权交还给 `clinical-supervisor`，由 supervisor 决定是否继续图谱推理和动态追问流程。
