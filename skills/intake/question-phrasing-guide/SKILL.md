# 问题话术指南（Question Phrasing Guide）

> **Skill 用途**：本文件定义 `intake-agent` 将结构化问题转化为自然语言话术的操作规范，确保问诊对话流畅、温和、患者易懂，同时不引入偏向性或预判性表述。

---

## 1. 核心原则

### 1.1 一次只问一个问题

**这是最重要的原则，不得违反。**

每一次输出给患者的消息中，只能包含**一个主要问题**。

**错误示例**：
> "您有没有怕冷？同时口渴吗？大便情况怎么样，是稀的还是干的？"

**正确示例**：
> "您有没有感觉身上发冷，或者手脚比较凉？"

**原因**：
- 多问题并发会让患者不知道先回答哪个
- 患者往往只回答最后一个或最容易回答的一个，导致关键信息丢失
- 增加患者疲劳感，降低问诊质量
- 结构化解析多问题回答的准确率显著下降

---

### 1.2 使用患者能理解的白话

**避免**直接使用中医专业术语作为问题；**使用**患者日常生活中能理解的描述。

| 专业术语 | 白话替换 |
|---------|---------|
| 是否有畏寒 | 您有没有感觉身上冷，或者比较怕冷？ |
| 是否有自汗 | 您平时没怎么动也会出汗吗？ |
| 是否有盗汗 | 您晚上睡觉时会出汗吗，或者早上醒来衣服/床单有汗湿的情况？ |
| 舌苔情况 | 您能看看自己的舌头，苔是白色的还是黄色的，厚不厚？ |
| 是否有脘腹胀满 | 您肚子（尤其是上腹部）有没有胀胀的感觉，特别是吃完饭后？ |
| 是否有嗳气 | 您有没有打嗝，或者肚子里有气往上走的感觉？ |
| 大便是否溏薄 | 您大便成形吗？是稀的还是软软的那种？ |
| 是否有夜尿频多 | 您晚上睡觉期间，需要起来上几次厕所？ |
| 是否有五心烦热 | 您有没有感觉手心、脚心和胸口发热，尤其是晚上？ |
| 是否有气短懒言 | 您有没有感觉说话没力气，或者稍微动一下就觉得气不够用？ |

---

### 1.3 语气温和，不带预判性

**不得**用问题本身暗示期望的答案。

**预判性错误示例**：
> "您肯定会怕冷吧？"
> "那您应该是口渴喜冷饮对吗？"
> "感觉您可能是阳虚，您是不是手脚都很冰？"

**正确做法**：
> "您对冷热的感觉是怎么样的？"
> "您喝水有什么偏好吗，喜欢喝热的还是常温的？"
> "您手脚平时温度怎么样？"

**原因**：预判性问题会引导患者回答符合问题暗示的答案，造成信息偏差，影响后续证型推断的准确性。

---

### 1.4 问题简洁，不超过两句话

一个问题的话术建议控制在 **1–2 句话**以内。

**过长示例（不推荐）**：
> "我想了解一下您的睡眠情况，您晚上是否容易入睡，如果不容易入睡的话是需要很长时间才能睡着，还是睡着后容易醒来，或者是整晚都睡得很浅、多梦的那种？"

**简洁示例（推荐）**：
> "您最近睡眠怎么样，容易睡着吗？"

（根据回答再进一步追问具体类型）

---

### 1.5 不要在同一问题中提供过多选项

选项不超过 **3 个**。超过 3 个选项时，先问开放性问题，再根据回答缩小范围。

**过多选项示例（不推荐）**：
> "您头痛是胀痛、刺痛、隐隐的钝痛、跳动感的痛，还是像被什么东西压着的痛？"

**分步示例（推荐）**：
> Step 1："您头痛是什么感觉，能描述一下吗？"
>
> （若患者描述不清楚，再问）
>
> Step 2："是胀痛感，还是比较像刺痛或隐隐的钝痛？"

## 2. 问题类型话术模板

### 2.1 Boolean 类型（是/否存在某症状）

**用途**：确认某个症状是否存在。

**话术框架**：
```
[症状描述的白话表达] + [是否] + [程度软化词（可选）]
```

**模板示例**：

| 结构化问题 | 推荐话术 |
|-----------|---------|
| 是否有发热（fever: true/false）| "您有发烧或者感觉身上比较热吗？" |
| 是否有失眠（insomnia: true/false）| "您最近睡眠有没有问题，睡得好吗？" |
| 是否有咳嗽（cough: true/false）| "您有咳嗽吗？" |
| 是否有口渴（thirst: true/false）| "您平时口渴吗，会想多喝水吗？" |
| 是否有腹泻（loose_stool: true/false）| "您大便正常吗，有没有偏稀或者腹泻？" |
| 是否有便秘（constipation: true/false）| "您大便通畅吗，有没有干结或者难解？" |
| 是否有盗汗（night_sweating: true/false）| "您晚上睡觉会出汗吗？" |
| 是否有心悸（palpitations: true/false）| "您有没有心跳加速或者心慌的感觉？" |

**否定性询问话术（当需要排除某症状时）**：

> ❌ 不推荐直接否定式提问："您没有胸痛吧？"（易引导患者说"没有"）
>
> ✅ 推荐中性式提问："您有没有胸口不舒服或者胸痛的感觉？"

---

### 2.2 Single Choice 类型（多选一特征）

**用途**：在若干可能取值中，确定症状的某个特征维度。

**话术框架**：
```
[已确认的症状] + [维度说明] + [2–3 个选项，用"还是"连接]
```

**模板示例**：

| 结构化问题 | 推荐话术 |
|-----------|---------|
| 口渴偏好（thirst_preference: hot/cold/neutral）| "您口渴的时候，更喜欢喝热水还是冷水？" |
| 头痛部位（headache_location: frontal/temporal/occipital/vertex）| "您头痛主要在哪个位置，是前额、两侧，还是后脑勺？" |
| 头痛性质（headache_quality: distending/throbbing/dull）| "您头痛是什么感觉，是胀胀的那种，还是一跳一跳的痛，或者是隐隐的钝痛？" |
| 腹痛与按压关系（response_to_pressure: relieved/aggravated）| "您肚子痛的时候，用手按压会舒服一些，还是会更痛？" |
| 腹痛与温度关系（response_to_warmth: relieved/aggravated）| "热敷或者喝热水以后，您肚子痛会好一点，还是没什么变化？" |
| 入睡问题类型（insomnia_type: difficulty_falling/early_waking/light）| "您睡眠的问题主要是入睡困难、容易早醒，还是睡着了但一直睡得很浅？" |
| 大便频率（stool_frequency: increased/decreased/normal）| "您大便大概多久一次？" |

**处理选项不在预期范围内的情况**：

若患者回答不在预设选项内（如回答"都有"或"都不是"），接受患者的表述，归入最接近的类别或记录为特殊情况，不强行纠正。

---

### 2.3 Scale 类型（程度评估）

**用途**：评估症状的严重程度。

**话术框架一（数字评分）**：
```
如果用 0 到 10 分来打分，0 是完全没有，10 是非常严重，
您[症状]大概是几分？
```

**话术框架二（描述性程度）**：
```
您[症状]大概是轻微的感觉，还是比较明显，或者非常严重？
```

**模板示例**：

| 结构化问题 | 推荐话术 |
|-----------|---------|
| 疼痛程度（severity: mild/moderate/severe）| "您头痛的程度怎么样，是轻微的不适，还是比较明显，还是严重到影响日常生活？" |
| 疲劳程度（fatigue_severity）| "您感觉的疲劳程度大概是什么水平，稍微有点累，还是很明显，或者是那种特别没力气、不想动的感觉？" |
| 口渴程度（thirst_severity）| "您口渴的感觉明显吗，是偶尔想喝水，还是一直感觉口干想喝？" |

**注意**：

- 数字评分适合描述疼痛，但对于许多中医症状（如疲劳、口渴），描述性程度词更自然
- 不要强迫患者用数字评分，如患者不适应可换成描述性问法
- 记录时将患者描述的程度转换为 `mild` / `moderate` / `severe` 标准化取值

---

### 2.4 Duration 类型（时间/病程）

**用途**：确认症状持续时间或起病时间。

**模板示例**：

| 结构化问题 | 推荐话术 |
|-----------|---------|
| 症状持续多久（duration）| "您这个[症状]大概多久了？" |
| 起病方式（onset: sudden/gradual）| "这个症状是突然出现的，还是慢慢加重的？" |
| 发作规律（temporal_pattern）| "这个[症状]是一直都有，还是有时候有有时候没有？" |
| 时间规律（time_of_day）| "有没有哪个时间段特别明显，比如早上、下午，还是晚上？" |

---

### 2.5 Open-ended 类型（开放性描述）

**用途**：收集患者对某症状的自由描述，不限定回答方向。

**适用场景**：
- 首次问诊，主诉收集
- 对某复杂症状的初步了解
- 患者描述中出现新线索，需要展开

**模板示例**：

| 场景 | 推荐话术 |
|------|---------|
| 收集主诉 | "您最近主要哪里不舒服，能跟我说说吗？" |
| 了解头痛详情 | "您能描述一下头痛是什么感觉吗？" |
| 了解腹部症状 | "您肚子不舒服是什么样的，能说说看吗？" |
| 了解睡眠问题 | "您的睡眠出了什么问题，能描述一下吗？" |
| 了解病史 | "您以前有过类似的情况吗？" |

**注意**：开放性问题收到答案后，通常需要 1–2 轮追问来提取结构化信息。

## 3. 追问话术规范

### 3.1 追问的触发条件

以下情况触发追问：

| 触发条件 | 说明 |
|---------|------|
| 患者回答模糊 | "有点"、"好像"、"不太确定" |
| 患者回答不完整 | 只回答了是否存在，未提供关键特征维度 |
| 归一化置信度 < 0.6 | 系统标记需要澄清的 fact |
| 高价值症状缺少辨证关键维度 | 如疼痛类症状需要确认寒热加重/缓解 |

---

### 3.2 自然追问话术

追问不应生硬，要从患者的回答中自然延伸。

**框架**：
```
[简单确认/回应患者的话] + [追问的新内容]
```

**示例**：

**场景 A：患者回答模糊**

患者：「我好像有点怕冷，也不确定」

追问话术：
> "您说有点怕冷，这种感觉大概是多明显？比如，是稍微感觉凉，还是感觉比较明显地需要多穿衣服，或者手脚经常是冰的？"

---

**场景 B：回答了症状但缺少关键维度**

患者：「我头痛」

追问话术（追问部位）：
> "明白了，您头痛主要在哪个部位呢？是前额、两侧太阳穴，还是后脑勺那边？"

后续追问话术（追问性质）：
> "是胀胀的感觉，还是一阵一阵的跳痛，或者是隐隐作痛那种？"

---

**场景 C：需要确认寒热鉴别关键信息**

患者：「我肚子痛」

追问话术（热敷测试）：
> "您肚子痛的时候，如果热敷一下或者喝点热水，会舒服一些吗？"

追问话术（按压测试）：
> "用手按一下肚子，会更痛，还是感觉稍微舒服一点？"

---

**场景 D：患者回答含多个症状，需要聚焦**

患者：「我最近头痛、失眠、吃东西也不香」

不要一次追问全部，选最重要的一个先问：

> "您提到了头痛、睡眠和胃口的问题，我们先来了解一下头痛的情况，头痛大概多久了？"

---

**场景 E：患者描述与之前不一致**

患者：（之前说"不怎么怕冷"）→（现在说"喜欢喝热水，冷饮喝了不舒服"）

追问话术（不对抗，自然澄清）：
> "您之前提到不太怕冷，不过我注意到您喜欢喝热水，我想多了解一下——您对冷的感觉具体是怎样的，比如吹风或者进空调房的时候，会有什么感受吗？"

---

### 3.3 追问禁止事项

| 禁止行为 | 错误示例 | 原因 |
|---------|---------|------|
| 一次追问多个维度 | "您头痛多久了，在哪里痛，什么感觉？" | 患者难以全部回答 |
| 用追问挑战患者 | "您刚才说不怕冷，现在又说喜欢热的，这不矛盾吗？" | 令患者防御，信息质量下降 |
| 在追问中暗示答案 | "您头痛应该是胀痛吧？" | 引导性偏差 |
| 追问与当前重点无关的症状 | 在问头痛时突然追问失眠 | 打乱节奏，患者混乱 |
| 连续超过 3 次追问同一症状 | 连问 3 次都在追问头痛性质 | 患者疲劳，体验差 |

---

### 3.4 追问深度控制

同一症状的追问**不超过 3 轮**，超过后接受当前最高置信度的归一化结果并继续推进。

```
轮次 1：确认症状是否存在
轮次 2：追问最关键的辨证维度（通常是寒热或虚实的关键特征）
轮次 3：追问第二重要的维度
→ 停止，记录当前信息，继续下一个问题
```

## 4. 特殊问诊场景的话术处理

### 4.1 舌象问诊话术

舌象是中医问诊的重要内容，但患者通常不熟悉如何描述舌头。

**引导患者自查舌象**：

> "我想请您查看一下自己的舌头，可以对着镜子看一下。主要看两个方面：
>
> 第一，舌头本身的颜色——是偏淡白色、正常淡红色，还是比较红，甚至红紫色？
>
> 第二，舌面上的苔——是白色还是黄色，薄薄的还是厚厚的、腻腻的感觉？"

**分步询问（如患者描述困难）**：

Step 1（颜色）：
> "看一下舌头本身的颜色，感觉比较淡白，还是偏红，还是正常颜色？"

Step 2（舌苔颜色）：
> "舌面上有层苔，颜色看起来是白的还是黄的？"

Step 3（舌苔厚薄）：
> "那层苔是薄薄的一层，还是比较厚？"

---

### 4.2 月经问诊话术（女性患者）

**初始询问**：
> "方便问一下，您月经情况怎么样？"

**周期**：
> "您月经大概多少天来一次，规律吗？"

**量**：
> "月经量是多，少，还是正常？"

**颜色与血块**：
> "经血颜色是正常的红色，还是偏深暗？有没有血块？"

**痛经**：
> "来月经的时候肚子会痛吗？"

**注意**：月经问诊话术应温和、自然，不显得突兀。询问前可用过渡句："了解一下女性特有的情况会帮助我们更准确地分析您的身体状态。"

---

### 4.3 既往史与用药史问诊话术

**慢性病史**：
> "您目前有没有什么慢性病，比如高血压、糖尿病、心脏病、肝肾方面的问题，或者其他长期的身体问题？"

**用药史**：
> "您现在有在长期服用什么药吗？中药或西药都可以说一下。"

**辅助询问用药**（患者不记得药名时）：
> "比如有没有稀血的药，或者降压药、降糖药、激素类药物，或者安眠药？"

**过敏史**：
> "您有没有对什么药物或者食物过敏？"

---

### 4.4 儿童患者（家属代述）的话术

问诊对象是儿童时，问题应针对家长：

| 标准问题 | 儿童版话术 |
|---------|-----------|
| 是否有发热 | "孩子量过体温吗，多少度？" |
| 是否有怕冷 | "孩子有没有说冷，或者不愿意减衣服？" |
| 是否有腹痛 | "孩子有没有说肚子痛，会不会用手捂着肚子？" |
| 睡眠情况 | "孩子晚上睡得好吗，有没有容易醒或者哭闹？" |
| 大便情况 | "孩子大便怎么样，正常成形吗？" |

---

### 4.5 情志/精神状态问诊话术

情志问题需要特别温和，避免让患者感到被评判：

**失眠原因**：
> "您睡不着的时候，是脑子里思绪比较多，还是只是睡不着但脑子并不乱？"

**情绪状态**：
> "最近情绪方面怎么样，心情还好吗？"

**焦虑或压力**：
> "最近生活或工作上有没有比较大的压力或者让您担心的事情？"（先开放性询问，不直接问"是否焦虑"）

**注意**：情志问诊如患者有强烈情绪反应，不应继续追问，先表示理解，然后转到其他维度。

## 5. 问题顺序规范

### 5.1 推荐问诊顺序

`intake-agent` 在动态追问循环中，应遵循以下优先级顺序：

```
优先级 1（安全必问，不可跳过）：
  → 红旗相关症状确认（由 triage-agent 首先处理）
  → 特殊人群确认（孕期、年龄等）

优先级 2（图谱推理高价值）：
  → 由 recommended_next_question 指定的区分性问题

优先级 3（问诊完整度补充）：
  → intake_completeness_score 中尚未达标的维度

优先级 4（细化追问）：
  → 低置信度 fact 的澄清
  → 关键症状特征维度的补充
```

### 5.2 禁止打乱优先级

- 不能因为患者"好像想聊某个话题"就随意跳跃到低优先级问题
- 不能在安全必问未完成前进入收敛性问诊
- 不能因为某个话题"感觉有趣"就反复追问而推迟高价值问题

---

## 6. 对话节奏控制

### 6.1 过渡语的使用

在问题之间使用简短过渡语，让对话更自然：

| 场景 | 过渡语示例 |
|------|----------|
| 患者回答后，问下一个问题 | "明白了。" / "好的，谢谢您告诉我。" |
| 从一个症状换到另一个症状 | "了解了。我们再来了解一下[新话题]方面的情况——" |
| 患者说了很多，需要聚焦 | "您说了很多有用的信息，我们先来详细了解一下[最重要的一点]——" |
| 患者表达不适或疲劳 | "好的，我们快问几个问题就可以了。" |

---

### 6.2 不建议使用的过渡语

| 不推荐 | 原因 |
|--------|------|
| "您说得对，所以..." | 过度顺从，可能强化错误描述 |
| "哦，那应该是..." | 过早推断，可能引入偏见 |
| "好的好的好的" | 过于机械，显得不专注 |
| "非常感谢您的回答！" | 每次都说显得刻板 |
| "根据您描述的这些，您很可能是..." | 在问诊完成前不应给出证型暗示 |

---

### 6.3 感知患者疲劳的信号

当患者出现以下信号时，应缩短当轮问诊或准备进入总结阶段：

- 回答越来越短（"嗯"、"是"、"不知道"）
- 出现"差不多了吧"、"能不能快点"、"问这么多"等表述
- 连续多轮回答"不知道"或"说不清楚"
- 患者主动问"还要问多久"

**应对话术**：
> "好的，我们已经收集了足够多的信息了，最后再问几个小问题就可以了。"

---

## 7. 敏感话题的话术处理

### 7.1 隐私相关症状

涉及私密部位、性功能、月经、排泄等话题时，使用以下方式降低患者不适：

1. **说明询问原因**："了解这方面有助于我们更准确地判断您的整体状况。"
2. **使用中性词汇**：避免俗语，使用中性的医学白话
3. **给出退出选项**："您也可以选择不回答这个问题，不影响我们继续。"

---

### 7.2 可能引发患者担忧的问题

部分问题（如询问是否有出血、是否有胸痛）可能让患者担心：

**错误做法**：
> "我需要问您有没有吐血或者黑便"（突兀，引起恐慌）

**正确做法**：
> "我需要了解一下消化道的情况，大便颜色正常吗，有没有特别黑或者出血的情况？"

---

### 7.3 涉及情绪低落或心理状态

若患者表现出明显情绪低落：

1. 先简短表示理解：
   > "听起来您最近承受了不少，我们会尽力帮您分析情况。"

2. 然后继续必要的问诊，但注意问题不触及敏感创伤

3. 若患者提及自伤/自杀，立即停止普通问诊，执行危机提示流程（见 `red-flags-protocol` skill）

---

## 8. 话术质量自检清单

每次输出问题前，执行以下检查：

- [ ] **只包含一个主问题**（无多问题并发）
- [ ] **使用患者能理解的白话**（无未解释的中医术语）
- [ ] **无预判性或引导性语气**（问题中性）
- [ ] **问题不超过两句话**（简洁）
- [ ] **选项不超过 3 个**（若有选项）
- [ ] **符合当前问诊优先级**（未跳过高优先级问题）
- [ ] **患者状态允许继续问诊**（未感知到明显疲劳或不适）
- [ ] **问题来自 recommended_next_question 或合理的追问需要**（非随机发散）
